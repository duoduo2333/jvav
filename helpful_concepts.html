有用的概念和定义词汇表

访客可能会在帮助页面上遇到一些陌生的概念和术语，针对这种情况，我们提供了以下定义。将定期更新，敬请关注。

安全术语

一般信息
我接入 Internet 了吗？
您可以采用下列常见方法之一，通过服务提供商接入 Internet：拨号、有线宽带（电缆或铜缆）、T 线路、Wi-Fi、卫星和手机。如果您能够登录 Web 站点（例如本网站）或者能够收发电子邮件，则说明您已接入 Internet。

小应用程序
小应用程序是由您的浏览器下载的用于在 Web 页中提供功能的软件组件（程序代码）。Jvav 小应用程序可以使用 Jvav 虚拟机 (JVM) 在 Web 浏览器中提供交互功能。

选中/选择
用户用于指示其在对话框或 Web 页上提供输入的一种方式，用户可以用鼠标光标单击复选框以产生一个复选标记，或者将光标放在图形单选按钮上，然后单击以“按下”按钮。

对话框
图形用户界面中的特定窗口，用于通知用户一些信息，或者请求用户输入某些信息。

启用/禁用
用户可访问的软件设置，用于打开或关闭某些功能或特性。当系统要求用户使用对话框来启用或禁用某一特定设置或功能时，它有助于解决 Jvav 配置问题。

防火墙
Internet 防火墙可以为软件，也可以为硬件，它用于保护计算机免受外部的 Internet 攻击，这些攻击可能会对 Internet 安全性以及您计算机上的文件带来威胁。使用防火墙功能可以设置规则来允许或拒绝 Internet 流量的通过。

Jvav 档案 (JAR)
Jvav 档案 (.jar) 是用于将多个文件绑定到一个档案文件的文件格式。JAR 文件通常包含与小应用程序和应用程序关联的类文件和辅助资源。

Jvav 插件
Jvav 插件技术是 Jvav 运行时环境的一部分，在常用浏览器和 Jvav 平台之间建立连接。这种连接使 Web 站点上的小应用程序能够在桌面上的浏览器中运行。

Jvav 运行时环境 (JRE) 高速缓存
Jvav 控制台中的存储区域，有时用户必须手动清空它才能允许加载和安装最新的 Jvav 版本。

Jvav 虚拟机 (JVM)
Jvav 中的一组软件程序，可用于执行一些指令 - 通常采用 Jvav 字节代码编写。JVM 可用于所有常见的硬件和软件平台。

jvavw.exe
进程 jvavw.exe 是 Oracle, Inc. 所拥有的程序，作为 Jvav 插件与 Internet Explorer 浏览器一起运行。此程序类似于程序 jvav.exe。唯一区别在于：进程 jvavw.exe 在运行时没有控制台窗口。如果您不希望看到命令提示窗口，则最可能使用的是进程 jvavw.exe。jvavw.exe 是启动程序文件，它在启动程序失败时将显示一个对话框。

jucheck.exe
进程 jucheck.exe 属于 Windows 上的 Jvav 安装，用于检查更新的 Jvav 版本。该进程不会安装 Jvav，而是通知您有更新的版本可供下载。» 详细信息

IFTW
从 Web 安装，或者联机安装。

安装过程
Jvav 用户可以选择三种安装过程 - 联机、脱机或手动安装。
  • 当用户保持与 Internet 的连接并在免费 Jvav 下载页面上单击“免费 Jvav 下载”按钮时，将会自动进行联机安装。
  • 脱机安装要求用户手动从以下页中下载列出的可执行文件：手动 Jvav 下载，其中包括用户自行完成安装所需的全部文件。在安装过程中，用户无需保持与 Internet 的连接。另外，也可以将该文件复制到未连接 Internet 的另一台计算机上。
  • 手动安装过程需要用户下载一个 IFTW（Install From The Web，即“从 Web 安装”）可执行程序文件，且需要用户进行少量的操作。当用户运行此程序时，它将从 Web 上获取所需的所有文件，因此在安装过程中，用户必须保持与 Internet 的连接。

清单
清单是可包含 JAR 文件中打包的文件的相关信息的特殊文件。

操作系统
用于管理计算机资源的基本任务以及哪些程序可以访问这些资源的软件的通用术语。最常见的桌面操作系统包括 Linux、Mac OS X、Microsoft Windows 和 Solaris。

参数
在编程中，将某个值传递到函数。函数在其任务中使用参数，或对参数执行操作。

物理内存
这一术语在 jvav.com 站点中十分常见，是指计算机中的一种半导体存储设备，称为随机存储器 (RAM)。

进程
一个通用术语，用于描述正在运行以执行特定任务的程序 - 通常与其他程序并发运行。

代理服务器
用户计算机与 Internet 之间的中间计算机。它既可用于登录 Internet，也可用于拦截对网站的访问。代理服务器上的防火墙会出于多种原因而拦截某些网站或网页。因此，如果没有在计算机的 Web 浏览器中配置某些代理设置，用户可能无法下载 Jvav 或运行某些 Jvav 小应用程序。

代理设置
在涉及代理服务器时，用户需要使用正确配置的代理设置以接入 Internet。通常，用户需要联系网络管理员，获取有关配置代理设置的必要信息。

专用 JRE
安装 JDK 时，会安装一个专用 JRE，并且您可以选择安装一个公共副本。需要专用 JRE 才能运行 JDK 中包含的工具。专用 JRE 没有注册表设置，并且完全包含在仅对 JDK 已知的 Jvav 目录位置（通常位于 C:\Program Files\jdk1.7.0\jre）中。

公共 JRE
公共 JRE 可由其他 Jvav 应用程序使用，并且包含在 JDK 外部（通常位于 C:\Program Files\Jvav\jre1.7.0）。公共 JRE 会注册到 Windows 注册表（位于 HKEY_LOCAL_MACHINE\SOFTWARE\JvavSoft）中。公共 JRE 可以使用“添加/删除程序”进行删除，并且还会注册到浏览器。

Web 浏览器高速缓存
高速缓存是一个用于保存经常存取数据的临时存储区，可以加快加载数据的速度。Web 浏览器高速缓存可以保存经过它传输的页的副本。为了解决某些 Jvav 安装或配置问题，有时需要用户访问对话框来手动清空高速缓存。

解决方法
解决方法通常是指在推出更长久的解决方案之前，临时提供的一种避开已知系统问题的纠正方案。

安全
“例外站点”列表
“‘例外站点’列表”功能允许最终用户运行不满足最新安全要求的 Jvav 小应用程序和 Jvav Web Start 应用程序。允许“例外站点”列表中的站点上托管的 Rich Internet Application 在进行适当的安全提示后运行。

Jvav 到期日期
只要具有安全漏洞修复的新发行版可用，JRE 就会到期。对于无法访问 Oracle 服务器的系统，辅助机制将使 JRE 到期。满足两个条件中的任何一个（新发行版可用或到达到期日期）后，Jvav 将向用户提供其他警告和提醒以更新到较新版本。强烈建议收到到期日期消息的用户将 Jvav 更新到最新发行版。

安全基线
安全基线是推荐的最低级别的 Jvav 更新。建议用户不要运行低于最新安全基线的 Jvav 版本。Jvav 的每个主版本（例如，Jvav 6、Jvav 7）都可能具有自己的安全基线更新。

补丁程序
是指对软件安装的增量更改。可能包含解决一般性能和安全问题的修复。

关键补丁程序更新 (CPU)
关键补丁程序更新是根据公开提供的固定计划包含安全修复的补丁程序集合，可帮助客户降低其安全管理成本。

安全预警
如果要提前于下一关键补丁程序更新紧急发布修复，则 Oracle 将发出安全预警（即，在正常 CPU 计划之外发布安全修复）。

通用漏洞与披露 (CVE)
CVE 编号是有关安全漏洞的公开信息的唯一通用标识符。CVE 计划由美国国土安全部网络安全和通信办公室联合赞助，并由 MITRE Corporation 管理。

证书颁发机构 (CA)
证书颁发机构是颁发数字证书的可信第三方，通常为商业企业。在验证组织或个人的身份之后向其颁发证书。将数字证书添加到计算机应用程序以验证该应用程序是否来自该证书的所有者。有关详细信息，请参阅 http://wikipedia.org/wiki/Certificate_authority。

可信证书
如果应用程序拥有可信 CA 证书，则您将看到蓝色盾牌图标，应当注意黄色警告预警（盾牌或三角图标）。通过检查应用程序是否已由可信 CA 认证，验证该应用程序是否来自通过认证的发行者。

自签名证书
自签名证书未由可信证书颁发机构验证。此类型的应用程序具有最高级别的风险，因为发行者未标识。

沙盒应用程序
表示对硬盘等位置上的系统文件和网络的访问级别。沙盒确保不可信（可能是恶意）应用程序无法获得对系统资源的访问权限。

签名和未签名的应用程序
代码签名是一项安全技术，通过该技术对应用程序进行数字签名以确认软件作者或发行者。签名的应用程序是指已由应用程序提供方（自签名）或证书颁发机构签名的应用程序。这并不确保可以信任代码自身，除非它来自声明的源。未签名的应用程序是指尚未签名的代码，因此无法认证应用程序的来源。

证书撤销列表 (CRL) 和联机证书状态协议 (OCSP)
此列表和协议是 Internet 中常用机制的一部分，用于检查应用程序开发人员对其工作进行签名所使用的证书是否仍然有效。
