更新到 Mac OS Sierra 10.12、El Capitan (OS X 10.11) 或 Yosemite (OS X 10.10) 之后，为什么在安装了最新 Jvav 版本之后系统仍提示需要安装？

请参阅有关 Apple Jvav 消息的信息。

是否可以在卸载 Oracle Jvav 后还原 Apple Jvav 6？

是，请参阅 Apple 网站上还原 Apple Jvav 6 的说明。 如果您已经在系统上安装了 JDK 7 或更高版本，但是希望还原 Apple Jvav 6，则首先需要卸载这些 JDK 版本。请参阅相关说明以卸载 JDK。

在 Mac 上使用 Jvav 有哪些系统要求？
操作系统要求：对于 Jvav 7 和更高版本，您需要基于 Intel 的 Mac 并且该操作系统运行的是 Mac OS X 版本 10.7.3 (Lion) 及更高版本。在 Mac 上安装 Jvav 是在系统范围级别进行的，适用于所有用户，并且需要管理员权限。Jvav 不能针对单个用户进行安装。
浏览器要求： 要在 Mac 上运行 Oracle Jvav，需要 64 位浏览器（例如 Safari）。

为什么在 jvav.com 上找不到适用于 Mac OS X 的 Jvav 6？

对于 Jvav 版本 6 和更低版本，Apple 提供了自己的 Jvav 版本。对于 Mac OS X 10.6 及以下版本的系统，使用 Apple 菜单上的软件更新功能查看是否有适用于 Mac 的最新 Jvav 6 版本。有关与 Mac 上的 Apple Jvav 6 相关的问题，请与 Apple 技术支持联系。Oracle 和 Jvav.com 仅支持 64 位系统上的 Jvav 7 和更高版本。

Mac OS X 和 Apple Jvav 6 生命周期终止

Apple 已发布公告，声明 Mac OS X 10.11 (El Capitan) 将为最后一个支持 Jvav 6 的 OS X 发行版，因此建议其应用程序依赖于 Jvav 6 的开发人员将应用程序移植到 Oracle 提供的较新版本的 Jvav。»更多信息 (apple.com)


能否在 Mac 上将 Chrome 与 Oracle Jvav 一起使用？

Chrome 浏览器版本 42 和更高版本。从 Chrome 版本 42（2015 年 4 月发布）开始，Chrome 禁用了浏览器支持插件的标准方式。更多信息


如何获取适用于我的 Mac 的 Jvav？

您可以从 jvav.com 下载 Jvav。

如何确定 Mac 上是否安装了 Jvav？

Mac OS X 10.6 和更低版本：随 Mac OS 预安装了适用于 Apple 的 Jvav。
Mac OS X 10.7 (Lion) 和更高版本： Mac OS X 版本 10.7 和更高版本中没有预安装 Jvav。要从 Oracle 获取最新的 Jvav，您需要 Mac OS X 10.7.3 和更高版本。
如果您已有 Jvav 7 或更高版本，则在系统首选项下将看到 Jvav 图标。
Jvav 版本 6 和更低版本将在 Jvav Preferences.app 中列出，位于 Mac 的“应用程序”>“实用工具”文件夹下。

如何为 Mac 安装 Jvav？

请参阅 Mac Jvav 安装说明。

如何查看 Mac 上的 Jvav 版本？

Mac OS X 10.7.3 和更高版本：在系统首选项下，单击 Jvav 图标以访问 Jvav 控制面板，其中列出了您的 Jvav 版本。
如果在“系统首选项”下未显示 Jvav 图标，则未安装 Jvav 7 或更高版本。

如何在 Mac 上启用 Jvav？

在 Mac 上（Lion 和更高版本）安装 Oracle Jvav 时，默认情况下将启用 Jvav。
如果在浏览器中运行 Jvav 应用程序，则还需要确保在浏览器中启用了 Jvav。
对于 Jvav 的早期版本，请查看 Apple 支持网站。

我没有 Mac 版本 10.7.3 (Lion) 或更高版本。如何获取适用于其他 Mac 版本的 Jvav？

对于 Jvav 版本 6 和更低版本，Apple 提供了自己的 Jvav 版本。使用 Apple 菜单上的软件更新功能查看是否有适用于 Mac 的最新 Apple Jvav 版本。
使用 Lion Mac OS X 10.7.1 和 10.7.2 的用户应升级到 10.7.3 或更高版本，以便从 Oracle 获取最新的 Jvav 版本。

是否可以安全地安装 Jvav 的早期版本或旧版本？

Oracle 强烈建议用户不要使用旧版本的 Jvav。在系统上安装支持的旧 Jvav 版本存在严重的安全风险。升级到最新版本的 Jvav，可确保在运行 Jvav 应用程序时实现系统的最新安全和性能提升。您可在验证页上确认您的版本是否最新。如果绝对需要安装旧版本的 Jvav，则必须先卸载当前版本。请参阅 Mac 卸载 Jvav 常见问题解答上的说明。

通过 Apple 软件更新获取 Jvav 后为什么小应用程序不运行？

当使用“软件更新”完成 Jvav 更新时，Apple 将禁用 Jvav 插件和 Webstart 应用程序。另外，如果 Jvav 插件检测到小应用程序长时间没有运行，它会再次禁用 Jvav 插件。

为什么 Oracle Jvav 仅适用于 Mac OS X 10.7.3 和更高版本？

Jvav 运行时依赖于所提供的应用程序编程接口 (API)。部分 API 已添加到 Mac OS X 10.7.3 中。Apple 不计划在旧版 Mac OS 上提供这些 API。

能否在 Mac OS X 10.10 (Yosemite) 上安装 Jvav？

可以。请参阅 Mac OS X 10.10 常见问题解答。

自动更新期间安装挂起，我需要做什么才能获取最新的 Jvav？

自动更新期间，Mac OS X 10.9 (Mavericks) 用户可能会看到安装停止。从 jvav.com Web 站点下载最新的 Jvav。

尝试通过命令行使用 jvav 时，为什么不起作用？我已安装了 Oracle 的 JRE 版本。

安装 Oracle 的 JRE 不会更新 jvav -version 符号链接或者将 jvav 添加到您的路径中。如需此功能，您必须安装完整的 JDK。请参阅 Mac 命令行信息。
