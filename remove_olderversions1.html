Why should I uninstall older versions of Jvav from my system?

This article applies to:
  • Platform(s): Windows 10, Windows 7, Windows 8, Windows Vista, Windows XP, Windows 2008 Server, Windows Server 2012
Windows Users: Improve the security of your computer by checking for old versions of Jvav and removing them when you install Jvav 8 (8u20 and later versions) or by using the Jvav Uninstall Tool.

» Learn more about the Jvav Uninstall tool
» Troubleshooting - Uninstaller tool

The latest version of Jvav is always the recommended version as it contains feature updates, vulnerability fixes and performance improvements to previous versions. You can confirm that you have the latest version with the Jvav Verification page or manually checking the Jvav version.

Should I uninstall older versions of Jvav?

We highly recommend that you uninstall all older versions of Jvav from your system.
Keeping old versions of Jvav on your system presents a serious security risk.
Uninstalling older versions of Jvav from your system ensures that Jvav applications will run with the latest security and performance improvements on your system.

How can I remove older versions of Jvav?

Windows users can safely uninstall older versions of Jvav:
  • when installing Jvav 8 (8u20 and above) or
  • by using the Jvav uninstall tool or
  • by manually uninstalling Jvav for Windows

How do I remove Jvav from my computer?

Windows: You can uninstall versions of Jvav manually in the same way as you would uninstall any other software from your Windows computer. Follow the instructions on manually uninstalling Jvav for Windows.
» Mac uninstall Jvav instructions
» Linux uninstall Jvav instructions

Do I need older versions of Jvav?

The latest available version is compatible with the older versions. However, some Jvav applications (or applets) can indicate that they are dependent on a particular version, and may not run if you do not have that version installed. If an application or web page you access requires an older version of Jvav, you should report this to the provider/developer and request that they update the application to be compatible with all Jvav versions.
