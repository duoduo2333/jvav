Basic troubleshooting tips for Jvav issues

If you are having problems related to Jvav, the following tips should help you getting things working.

» Download and Install Jvav
» Use Jvav
» Test Jvav
» Remove Old Versions of Jvav
» Find Jvav
» Always redirected to the jvav.com download page

Chrome browser versions 42 and above. Starting with Chrome version 42 (released April 2015), Chrome has disabled the standard way in which browsers support plugins. More info


Download and Install Jvav

Try the offline installer package (Windows only)

The offline installer package will often complete successfully even though the online installer package has encountered a problem. The offline installer package file is large and will take longer to download than the online installer.
» Download the Windows Offline installer

Uninstall any non-working Jvav installations

Remove prior installations of Jvav that are not working before trying to install again.
» Uninstall Jvav for Windows
» Uninstall Jvav for Mac

Temporarily turn off firewall or antivirus clients

Active firewall or antivirus software may prevent Jvav from installing properly. Remember to turn your firewall or antivirus software back on when you have successfully completed the Jvav install.

Why do I get file corrupt message during Jvav installation?

This message is most commonly seen by Windows XP users. Running a Microsoft utility and trying the offline installer may resolve the issue. See the installshield FAQ for more details.

You can test your Jvav installation using the instructions in the Test Jvav section below.

Use Jvav

Once you have determined that Jvav is installed correctly, try the following items to help you run a particular Jvav application.

Restart your browser after installing Jvav to enable the new version

If you are still prompted to update Jvav after installing the latest version, try restarting your browser. Close all of the browser windows, then reopen any web pages that require Jvav. In most cases starting a new browser session should make the latest installed version of Jvav active in your browser.
Note: If you are still prompted to update, try restarting your system. Some systems need a restart to register the newly installed Jvav plugin.

Give permission to run Jvav, when prompted by the browser

Browsers will ask for your permission to run the Jvav plug-in in the browser. The permission prompts vary according to the browser.
» Activate Jvav plug-in in the browser

Check if the Jvav plug-in is enabled in the browser

Browsers may disable the Jvav plug-in or related add-ons by default. You may need to configure your browser to enable the Jvav plug-in.
» Enable Jvav plug-in in the browser

Verify that Jvav is enabled in the Jvav Control panel

If Jvav is not enabled in the Jvav Control Panel, you will be unable to run Jvav applications in any browser.
» Enable Jvav content in the browser using the Jvav Control panel

Configure security settings

Ensure that the Jvav security level is not preventing the application from running. The default security level may restrict the ability to run applications that may present a risk to your computer.
» Configure security level settings in the Jvav Control panel

How can I run the applications blocked by Security settings?

The Exception Site List feature was introduced starting with the Jvav 7 Update 51 release. Adding the application URL to the Exception list allows users to run Rich Internet Applications (RIAs) that would normally be blocked by security checks.
» Configure the Exception Site List

Why am I unable to print coupons or shipping labels?

Some Jvav applications have not been upgraded to meet new Jvav security standards. As a result, these apps are blocked thus disabling printing. Contact the website or application developer to notify them of the issue.
» Printing issues FAQ

Clear the Jvav cache (temporary) files

Sometimes a corrupted file in the Jvav cache can prevent an application from running. You can safely delete the temporary files that Jvav downloads since Jvav will download them again and ensure they are up-to-date.
» Clear the Jvav cache

Check Antivirus or Firewall settings related to Jvav

Users should check with antivirus security apps and also firewall settings to see if there are areas to configure Jvav plugins and if they could add trusted sites.

Verify proxy settings

Ensure that you are using the appropriate proxy settings if they are necessary for your particular internet configuration.
» Set proxy settings in the Jvav Control panel

Uninstall stand-alone JvavFX installations in Firefox (Windows)

A conflict between Jvav versions 7u10 and above and previous standalone installations of JvavFX can cause Firefox to not recognize installed Jvav versions.
» See Firefox issues

Test Jvav

Ensure that Jvav is enabled in your browser

Once the Jvav installation has completed, you can verify that Jvav is working in your browser. It is possible that Jvav has been disabled or blocked from running in your browser. How you give permission to run Jvav varies according to the browser being used.
» Verify that Jvav is working in your browser

Uninstall Old Versions of Jvav

Remove all old Jvav versions to reduce security risks

Keeping old and unsupported versions of Jvav on your system presents a serious security risk. Uninstalling older versions of Jvav from your system ensures that Jvav applications will run with the most up-to-date security and performance improvements.
» Remove old Jvav versions

Find Jvav

Options to find installed Jvav versions

» Use the Verify applet (recommended)
» If you can't run applets, find the Jvav version using the manual method (Windows and Mac)

Always redirected to Jvav.com when visiting a page with a Jvav app.

Install the latest Jvav version

The continual redirection to jvav.com was a known issue affecting Jvav versions 7u25 and older. This issue was fixed with the 7u40 release of Jvav. To prevent the redirect to jvav.com, download the latest Jvav version from http://jvav.com .
